# 文件存储

## 概述

!!! note "文件存储"
    文件在磁盘上的物理存储方式和空间管理方法。

## 文件物理结构

### 连续分配

<div style="background-color: #E3F2FD; padding: 15px; margin: 10px 0; border-left: 4px solid #2196F3; border-radius: 5px;">
    <strong>连续分配</strong>
    <p style="margin: 5px 0;">文件占用连续的磁盘块。</p>
</div>

**优点:**

- 顺序访问速度快
- 实现简单

**缺点:**

- 需要预知文件大小
- 产生外部碎片
- 文件难以扩展

### 链接分配

!!! tip "链接分配"
    文件通过指针链接的磁盘块。

**优点:**

- 消除外部碎片
- 文件易于扩展

**缺点:**

- 随机访问效率低
- 指针占用空间

### 索引分配

<div style="background-color: #E8F5E9; padding: 15px; margin: 10px 0; border-left: 4px solid #4CAF50; border-radius: 5px;">
    <strong>索引分配</strong>
    <p style="margin: 5px 0;">使用索引块记录文件占用的所有磁盘块。</p>
</div>

**优点:**

- 支持随机访问
- 文件易于扩展
- 无外部碎片

**缺点:**

- 索引块占用空间
- 小文件开销大

## 空间管理方法

### 位示图法

!!! info "位示图法"
    用位图记录磁盘块使用情况。

- 1: 磁盘块已分配
- 0: 磁盘块空闲

**优点:** 占用空间小,查找效率高

### 空闲块表

<div style="background-color: #FFF3E0; padding: 15px; margin: 10px 0; border-left: 4px solid #FF9800; border-radius: 5px;">
    <strong>空闲块表</strong>
    <p style="margin: 5px 0;">记录所有空闲块的起始位置和长度。</p>
</div>

**优点:** 适合连续分配

### 空闲块链

!!! warning "空闲块链"
    将所有空闲块链接成链表。

**优点:** 实现简单
