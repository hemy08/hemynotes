# 文件目录

## 概述

!!! note "文件目录"
    文件目录是文件系统中用于组织和管理文件的数据结构,实现文件的按名存取。

## 目录结构

### 单级目录

<div style="background-color: #E3F2FD; padding: 15px; margin: 10px 0; border-left: 4px solid #2196F3; border-radius: 5px;">
    <strong>单级目录</strong>
    <p style="margin: 5px 0;">整个文件系统只有一个目录。</p>
</div>

**特点:**

- 实现简单
- 文件不能重名
- 查找效率低

### 两级目录

!!! tip "两级目录"
    分为系统目录和用户目录。

**特点:**

- 不同用户文件可重名
- 查找效率提高
- 缺乏灵活性

### 多级目录(树形目录)

<div style="background-color: #E8F5E9; padding: 15px; margin: 10px 0; border-left: 4px solid #4CAF50; border-radius: 5px;">
    <strong>多级目录</strong>
    <p style="margin: 5px 0;">目录形成树形结构。</p>
</div>

**特点:**

- 层次清晰
- 便于组织
- 允许重名

## 目录项

!!! info "目录项"
    目录中的每个条目,描述一个文件或子目录。

**内容:**

- 文件名
- 文件控制块指针
- 文件类型

## 路径

<div style="background-color: #FFF3E0; padding: 15px; margin: 10px 0; border-left: 4px solid #FF9800; border-radius: 5px;">
    <strong>路径类型</strong>
</div>

### 绝对路径

- 从根目录开始的完整路径
- 示例: /home/user/file.txt

### 相对路径

!!! warning "相对路径"
    从当前目录开始的路径。

- 示例: ./docs/file.txt
