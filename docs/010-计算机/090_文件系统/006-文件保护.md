# 文件保护

## 概述

!!! note "文件保护"
    防止文件被非法访问、修改或破坏,保证文件的安全性和完整性。

## 保护目标

<div style="background-color: #E3F2FD; padding: 15px; margin: 10px 0; border-left: 4px solid #2196F3; border-radius: 5px;">
    <strong>保护目标</strong>
    <ul style="margin: 5px 0;">
        <li><strong>保密性</strong>: 防止未授权访问</li>
        <li><strong>完整性</strong>: 防止非法修改</li>
        <li><strong>可用性</strong>: 保证合法访问</li>
    </ul>
</div>

## 访问控制

### 访问控制矩阵

!!! tip "访问控制矩阵"
    行表示用户,列表示文件,矩阵元素表示访问权限。

**优点:** 直观清晰

**缺点:** 空间开销大

### 访问控制表(ACL)

<div style="background-color: #E8F5E9; padding: 15px; margin: 10px 0; border-left: 4px solid #4CAF50; border-radius: 5px;">
    <strong>访问控制表</strong>
    <p style="margin: 5px 0;">每个文件关联一个访问控制表。</p>
</div>

**格式:** 文件 -> [(用户1, 权限1), (用户2, 权限2), ...]

### 权限位

!!! info "权限位"
    每个文件设置权限位。

**Unix权限:**

- r: 读权限
- w: 写权限
- x: 执行权限

**示例:** wxr-xr--

## 用户分类

<div style="background-color: #FFF3E0; padding: 15px; margin: 10px 0; border-left: 4px solid #FF9800; border-radius: 5px;">
    <strong>用户分类</strong>
</div>

- **文件主**: 文件创建者
- **同组用户**: 同一用户组
- **其他用户**: 其他所有用户

## 保护机制

### 口令保护

!!! warning "口令保护"
    为文件设置访问口令。

**优点:** 简单易实现

**缺点:** 口令易泄露

### 加密保护

<div style="background-color: #F3E5F5; padding: 15px; margin: 10px 0; border-left: 4px solid #9C27B0; border-radius: 5px;">
    <strong>加密保护</strong>
    <p style="margin: 5px 0;">对文件内容加密存储。</p>
</div>

**优点:** 安全性高

**缺点:** 性能开销
