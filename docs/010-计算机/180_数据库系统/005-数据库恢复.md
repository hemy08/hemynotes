# 数据库恢复

## 概述

!!! note "数据库恢复"
    将数据库从故障状态恢复到正确状态。

## 故障类型

<div style="background-color: #E3F2FD; padding: 15px; margin: 10px 0; border-left: 4px solid #2196F3; border-radius: 5px;">
    <strong>故障类型</strong>
    <ul style="margin: 5px 0;">
        <li><strong>事务故障</strong>: 事务执行错误</li>
        <li><strong>系统故障</strong>: 系统崩溃</li>
        <li><strong>介质故障</strong>: 磁盘损坏</li>
    </ul>
</div>

## 恢复技术

### 日志文件

!!! tip "日志文件"
    记录事务对数据库的更新操作。

**内容:**

- 事务标识
- 操作类型
- 更新前数据
- 更新后数据

### 检查点

<div style="background-color: #E8F5E9; padding: 15px; margin: 10px 0; border-left: 4px solid #4CAF50; border-radius: 5px;">
    <strong>检查点技术</strong>
    <p style="margin: 5px 0;">定期保存数据库状态。</p>
</div>

**优点:** 减少恢复时间

### REDO和UNDO

!!! info "重做和撤销"
    - **REDO**: 重做已提交事务
    - **UNDO**: 撤销未提交事务

## 恢复策略

<div style="background-color: #FFF3E0; padding: 15px; margin: 10px 0; border-left: 4px solid #FF9800; border-radius: 5px;">
    <strong>恢复策略</strong>
</div>

- 利用日志进行恢复
- 利用检查点加速恢复
- 利用备份进行恢复
