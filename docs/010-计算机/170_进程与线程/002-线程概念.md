# 线程概念

## 概述

!!! note "线程"
    线程是进程内的执行单元,是CPU调度的基本单位。

## 线程特点

<div style="background-color: #E3F2FD; padding: 15px; margin: 10px 0; border-left: 4px solid #2196F3; border-radius: 5px;">
    <strong>线程特点</strong>
    <ul style="margin: 5px 0;">
        <li><strong>轻型实体</strong>: 创建开销小</li>
        <li><strong>独立调度</strong>: CPU调度基本单位</li>
        <li><strong>共享资源</strong>: 共享进程资源</li>
        <li><strong>可并发执行</strong>: 多线程并发</li>
    </ul>
</div>

## 线程实现方式

### 用户级线程

!!! tip "用户级线程"
    线程管理在用户空间实现。

**优点:**

- 线程切换快
- 不需要内核支持

**缺点:**

- 一个线程阻塞,整个进程阻塞
- 不能利用多CPU

### 内核级线程

<div style="background-color: #E8F5E9; padding: 15px; margin: 10px 0; border-left: 4px solid #4CAF50; border-radius: 5px;">
    <strong>内核级线程</strong>
    <p style="margin: 5px 0;">线程管理由内核实现。</p>
</div>

**优点:**

- 可利用多CPU
- 一个线程阻塞不影响其他线程

**缺点:**

- 线程切换开销大

## 线程状态

!!! info "线程状态"
    - **就绪**: 等待CPU
    - **运行**: 正在执行
    - **阻塞**: 等待事件
