# 中断系统

## 概述

中断是计算机系统中的重要机制,用于处理突发事件和实现I/O操作。

!!! note "中断定义"
    中断是指CPU暂停当前程序的执行,转去处理紧急事件,处理完后再返回原程序继续执行的过程。

## 中断的作用

<div style="background-color: #E3F2FD; padding: 15px; margin: 10px 0; border-left: 4px solid #2196F3; border-radius: 5px;">
    <strong>中断的主要作用</strong>
    <ul style="margin: 5px 0;">
        <li>实现CPU与I/O设备并行工作</li>
        <li>处理硬件故障和软件错误</li>
        <li>实现人机交互</li>
        <li>支持多道程序设计</li>
        <li>实现实时处理</li>
    </ul>
</div>

## 中断的类型

### 1. 按中断源分类

<div style="background-color: #E8F5E9; padding: 15px; margin: 10px 0; border-left: 4px solid #4CAF50; border-radius: 5px;">
    <strong>硬中断(外部中断)</strong>
    <p style="margin: 5px 0;">由外部设备产生的中断。</p>
</div>

**类型:**

- I/O中断: I/O设备完成操作
- 时钟中断: 定时器到期
- 外部信号中断: 外部硬件信号

<div style="background-color: #FFF3E0; padding: 15px; margin: 10px 0; border-left: 4px solid #FF9800; border-radius: 5px;">
    <strong>软中断(内部中断)</strong>
    <p style="margin: 5px 0;">由程序执行产生的中断。</p>
</div>

**类型:**

- 程序中断: 除零、溢出
- 陷阱中断: 系统调用
- 异常中断: 非法指令

### 2. 按可屏蔽性分类

!!! tip "可屏蔽中断"
    可以被CPU禁止的中断。

- 可通过中断屏蔽字控制
- 用于非紧急事件

!!! warning "不可屏蔽中断(NMI)"
    不能被CPU禁止的中断。

- 用于紧急事件
- 如电源故障、内存校验错误

## 中断处理过程

```mermaid
sequenceDiagram
    participant CPU as CPU
    participant Device as 中断源
    participant Handler as 中断处理程序
    
    Device->>CPU: 发出中断请求
    CPU->>CPU: 判断是否响应
    CPU->>CPU: 保护现场
    CPU->>Handler: 转向中断处理程序
    Handler->>Handler: 处理中断
    Handler->>CPU: 恢复现场
    CPU->>CPU: 返回原程序
```

### 1. 中断请求

<div style="border: 2px solid #4CAF50; padding: 10px; margin: 10px 0; border-radius: 5px;">
    <strong>中断请求</strong>
    <p style="margin: 5px 0;">中断源向CPU发出中断请求信号。</p>
</div>

### 2. 中断判优

<div style="border: 2px solid #2196F3; padding: 10px; margin: 10px 0; border-radius: 5px;">
    <strong>中断判优</strong>
    <p style="margin: 5px 0;">当多个中断同时发生时,选择优先级最高的中断。</p>
</div>

**判优方式:**

- 硬件判优: 使用中断控制器
- 软件判优: 软件查询

### 3. 中断响应

<div style="border: 2px solid #FF9800; padding: 10px; margin: 10px 0; border-radius: 5px;">
    <strong>中断响应</strong>
    <p style="margin: 5px 0;">CPU响应中断,进入中断处理。</p>
</div>

**响应条件:**

- CPU开中断
- 当前指令执行完
- 无更高优先级中断

### 4. 保护现场

<div style="border: 2px solid #9C27B0; padding: 10px; margin: 10px 0; border-radius: 5px;">
    <strong>保护现场</strong>
    <p style="margin: 5px 0;">保存当前程序的执行状态。</p>
</div>

**保存内容:**

- 程序计数器(PC)
- 状态寄存器(PSW)
- 通用寄存器

### 5. 中断处理

<div style="border: 2px solid #E91E63; padding: 10px; margin: 10px 0; border-radius: 5px;">
    <strong>中断处理</strong>
    <p style="margin: 5px 0;">执行中断处理程序。</p>
</div>

### 6. 恢复现场

<div style="border: 2px solid #00BCD4; padding: 10px; margin: 10px 0; border-radius: 5px;">
    <strong>恢复现场</strong>
    <p style="margin: 5px 0;">恢复原程序的执行状态。</p>
</div>

## 中断嵌套

!!! success "中断嵌套"
    在处理中断时,可以响应更高优先级的中断。

```mermaid
graph TB
    A[主程序] --> B[中断1]
    B --> C[中断2<br/>更高优先级]
    C --> B
    B --> A
```

**条件:**

- 新中断优先级更高
- 当前中断处理开中断

<div style="background-color: #F3E5F5; padding: 15px; margin: 10px 0; border-left: 4px solid #9C27B0; border-radius: 5px;">
