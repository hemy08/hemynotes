# 总线系统

## 概述

总线(Bus)是连接计算机各部件的公共通信线路,是计算机系统中传输信息的公共通路。

!!! note "总线定义"
    总线是连接计算机各部件的一组公共通信线路,用于在各部件之间传输信息。

## 总线的分类

### 1. 按功能分类

<div style="background-color: #E3F2FD; padding: 15px; margin: 10px 0; border-left: 4px solid #2196F3; border-radius: 5px;">
    <strong>数据总线(Data Bus)</strong>
    <p style="margin: 5px 0;">用于传输数据信息,双向总线。</p>
</div>

**特点:**

- 双向传输
- 宽度决定数据传输能力
- 与字长相关

<div style="background-color: #E8F5E9; padding: 15px; margin: 10px 0; border-left: 4px solid #4CAF50; border-radius: 5px;">
    <strong>地址总线(Address Bus)</strong>
    <p style="margin: 5px 0;">用于传输地址信息,单向总线。</p>
</div>

**特点:**

- 单向传输
- 宽度决定寻址能力
- 地址线数 = log₂(存储单元数)

<div style="background-color: #FFF3E0; padding: 15px; margin: 10px 0; border-left: 4px solid #FF9800; border-radius: 5px;">
    <strong>控制总线(Control Bus)</strong>
    <p style="margin: 5px 0;">用于传输控制信号和状态信号。</p>
</div>

**常见控制信号:**

- 读/写信号
- 中断请求
- 总线请求
- 时钟信号

### 2. 按位置分类

!!! tip "按位置分类"
    总线按所在位置可分为:

#### 片内总线

<div style="border: 2px solid #4CAF50; padding: 10px; margin: 10px 0; border-radius: 5px;">
    <strong>片内总线</strong>
    <p style="margin: 5px 0;">芯片内部的总线,连接芯片内部各部件。</p>
</div>

**示例:**

- CPU内部总线
- 寄存器之间的连接

#### 系统总线

<div style="border: 2px solid #2196F3; padding: 10px; margin: 10px 0; border-radius: 5px;">
    <strong>系统总线</strong>
    <p style="margin: 5px 0;">连接CPU、存储器、I/O接口的总线。</p>
</div>

**示例:**

- PCI总线
- PCI Express总线

#### 通信总线

<div style="border: 2px solid #FF9800; padding: 10px; margin: 10px 0; border-radius: 5px;">
    <strong>通信总线</strong>
    <p style="margin: 5px 0;">连接计算机系统之间的总线。</p>
</div>

**示例:**

- USB总线
- RS-232总线
- CAN总线

### 总线带宽计算

<div style="background-color: #F3E5F5; padding: 15px; margin: 10px 0; border-left: 4px solid #9C27B0; border-radius: 5px;">
    <strong>总线带宽计算公式</strong>
    <p style="margin: 5px 0;">总线带宽 = 总线宽度 × 总线频率 / 8</p>
</div>

**示例:**

- 总线宽度: 64位
- 总线频率: 100MHz
- 总线带宽: 64 × 100MHz / 8 = 800MB/s

## 总线结构

### 1. 单总线结构

!!! note "单总线结构"
    所有部件连接在同一条总线上。

```mermaid
graph LR
    A[CPU] --> B[系统总线]
    C[存储器] --> B
    D[I/O设备] --> B
```

**优点:**

- 结构简单
- 成本低

**缺点:**

- 性能受限
- 总线竞争

### 2. 双总线结构

!!! tip "双总线结构"
    使用两条总线分离存储器和I/O。

```mermaid
graph TB
    A[CPU] --> B[存储总线]
    B --> C[存储器]
    A --> D[I/O总线]
    D --> E[I/O设备]
```

**优点:**

- 性能较好
- 减少竞争

**缺点:**

- 结构复杂
- 成本较高

### 3. 多总线结构

!!! success "多总线结构"
    使用多条总线提高性能。

```mermaid
graph TB
    A[CPU] --> B[前端总线]
    B --> C[北桥芯片]
    C --> D[内存总线]
    D --> E[存储器]
    C --> F[PCI Express总线]
    F --> G[显卡]
    C --> H[南桥芯片]
    H --> I[PCI总线]
    I --> J[I/O设备]
```

**优点:**

- 性能高
- 扩展性好

**缺点:**

- 结构复杂
- 成本高

## 总线控制

### 1. 总线仲裁

<div style="background-color: #FCE4EC; padding: 15px; margin: 10px 0; border-left: 4px solid #E91E63; border-radius: 5px;">
    <strong>总线仲裁</strong>
    <p style="margin: 5px 0;">决定哪个设备获得总线控制权。</p>
</div>

**仲裁方式:**

- 集中式仲裁: 由总线控制器仲裁
- 分布式仲裁: 各设备自行仲裁

### 2. 总线定时

<div style="background-color: #E0F2F1; padding: 15px; margin: 10px 0; border-left: 4px solid #00BCD4; border-radius: 5px;">
    <strong>总线定时</strong>
    <p style="margin: 5px 0;">控制总线操作的时序。</p>
</div>

**定时方式:**

- 同步定时: 使用时钟信号
- 异步定时: 使用握手信号

## 常见总线标准

!!! warning "常见总线标准"
    现代计算机使用的总线标准。

### 1. PCI总线

- 32位/64位
- 33MHz/66MHz
- 带宽: 133MB/s - 533MB/s

### 2. PCI Express总线

- 串行传输
- 多通道(x1, x4, x8, x16)
- 高带宽: 最高32GB/s

### 3. USB总线

- 通用串行总线
- 支持热插拔
- 多种速度: 1.5Mbps - 40Gbps
