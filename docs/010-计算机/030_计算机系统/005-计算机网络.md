# 计算机网络

## 概述

计算机网络是将地理上分散的、具有独立功能的计算机系统,通过通信设备和线路连接起来,在网络软件支持下实现数据通信和资源共享的系统。

!!! note "计算机网络定义"
    计算机网络是计算机技术与通信技术相结合的产物,实现了资源共享和信息传递。

## 网络的功能

<div style="background-color: #E3F2FD; padding: 15px; margin: 10px 0; border-left: 4px solid #2196F3; border-radius: 5px;">

    <ul style="margin: 5px 0;">

    </ul>
</div>

### 按拓扑结构分类

!!! tip "网络拓扑结构"
    网络中节点的连接方式。

#### 1. 星型拓扑

<div style="background-color: #E8F5E9; padding: 10px; margin: 10px 0; border-left: 4px solid #4CAF50;">
    <strong>星型拓扑</strong>
    <p style="margin: 5px 0;">所有节点连接到中心节点。</p>
</div>

**优点:**

- 结构简单
- 易于扩展
- 故障隔离容易

**缺点:**

- 中心节点负担重
- 中心节点故障影响全网

#### 2. 环型拓扑

<div style="background-color: #FFF3E0; padding: 10px; margin: 10px 0; border-left: 4px solid #FF9800;">
    <strong>环型拓扑</strong>
    <p style="margin: 5px 0;">节点首尾相连形成环。</p>
</div>

**优点:**

- 结构简单
- 传输延迟确定

**缺点:**

- 可靠性差
- 扩展困难

#### 3. 总线型拓扑

<div style="background-color: #F3E5F5; padding: 10px; margin: 10px 0; border-left: 4px solid #9C27B0;">
    <strong>总线型拓扑</strong>
    <p style="margin: 5px 0;">所有节点连接到一条总线上。</p>
</div>

**优点:**

- 结构简单
- 成本低

**缺点:**

- 可靠性差
- 性能受限

## 网络协议

!!! info "网络协议"
    网络协议是计算机网络中进行数据交换的规则、标准或约定。

### OSI七层模型

```mermaid
graph TB
    A[应用层<br/>Application] --> B[表示层<br/>Presentation]
    B --> C[会话层<br/>Session]
    C --> D[传输层<br/>Transport]
    D --> E[网络层<br/>Network]
    E --> F[数据链路层<br/>Data Link]
    F --> G[物理层<br/>Physical]
```

### TCP/IP四层模型

<div style="background-color: #FCE4EC; padding: 15px; margin: 10px 0; border-left: 4px solid #E91E63; border-radius: 5px;">
    <strong>TCP/IP模型</strong>
</div>

**各层功能:**

- **应用层**: 提供网络应用服务(HTTP, FTP, SMTP)
- **传输层**: 提供端到端通信(TCP, UDP)
- **网际层**: 提供路由选择(IP, ICMP)
- **网络接口层**: 提供物理传输

## IP地址与子网

!!! info "IP地址"
    网络中设备的唯一标识符。

### IPv4地址

<div style="background-color: #E3F2FD; padding: 15px; margin: 10px 0; border-left: 4px solid #2196F3; border-radius: 5px;">
    <strong>IPv4地址</strong>
    <p style="margin: 5px 0;">32位地址,通常表示为4个十进制数,如192.168.1.1。</p>
</div>

**分类:**

- **A类**: 1.0.0.0 ~ 126.255.255.255 (大型网络)
- **B类**: 128.0.0.0 ~ 191.255.255.255 (中型网络)
- **C类**: 192.0.0.0 ~ 223.255.255.255 (小型网络)
- **D类**: 224.0.0.0 ~ 239.255.255.255 (多播地址)
- **E类**: 240.0.0.0 ~ 255.255.255.255 (保留地址)

**特殊地址:**

- 0.0.0.0: 本机
- 127.0.0.1: 本地回环地址
- 255.255.255.255: 广播地址

### IPv6地址

<div style="background-color: #E8F5E9; padding: 15px; margin: 10px 0; border-left: 4px solid #4CAF50; border-radius: 5px;">
    <strong>IPv6地址</strong>
    <p style="margin: 5px 0;">128位地址,解决IPv4地址枯竭问题。</p>
</div>

**特点:**

- 地址空间巨大(2^128个地址)
- 表示为8组16进制数,如2001:0db8:85a3:0000:0000:8a2e:0370:7334
- 支持自动配置
- 内置安全性

### 子网掩码

!!! tip "子网掩码"
    用于划分网络地址和主机地址。

**作用:**

- 确定网络部分和主机部分
- 支持子网划分
- 提高IP地址利用率

**CIDR表示法:** 192.168.1.0/24

## 网络协议详解

### TCP协议

<div style="background-color: #FFF3E0; padding: 15px; margin: 10px 0; border-left: 4px solid #FF9800; border-radius: 5px;">
    <strong>TCP(传输控制协议)</strong>
    <p style="margin: 5px 0;">面向连接的、可靠的传输层协议。</p>
</div>

**特点:**

- 面向连接: 三次握手建立连接
- 可靠传输: 确认、重传、流量控制
- 有序传输: 数据按序到达
- 全双工通信

**三次握手:**

```mermaid
sequenceDiagram
    participant Client as 客户端
    participant Server as 服务器

    Client->>Server: SYN=1, seq=x
    Server->>Client: SYN=1, ACK=1, seq=y, ack=x+1
    Client->>Server: ACK=1, seq=x+1, ack=y+1
```

**四次挥手:**

```mermaid
sequenceDiagram
    participant Client as 客户端
    participant Server as 服务器

    Client->>Server: FIN=1, seq=u
    Server->>Client: ACK=1, seq=v, ack=u+1
    Server->>Client: FIN=1, seq=w
    Client->>Server: ACK=1, seq=u+1, ack=w+1
```

### UDP协议

!!! warning "UDP(用户数据报协议)"
    无连接的、不可靠的传输层协议。

**特点:**

- 无连接: 不需要建立连接
- 不可靠: 不保证数据到达
- 无序: 数据可能乱序到达
- 高效: 开销小,速度快

**应用场景:**

- 实时音视频传输
- DNS查询
- 在线游戏

### HTTP协议

<div style="background-color: #F3E5F5; padding: 15px; margin: 10px 0; border-left: 4px solid #9C27B0; border-radius: 5px;">
    <strong>HTTP(超文本传输协议)</strong>
    <p style="margin: 5px 0;">应用层协议,用于Web数据传输。</p>
</div>

**版本:**

- **HTTP/1.0**: 短连接,每次请求建立新连接
- **HTTP/1.1**: 持久连接,管道化请求
- **HTTP/2**: 多路复用,头部压缩,服务器推送
- **HTTP/3**: 基于QUIC协议,使用UDP

**请求方法:**

- GET: 获取资源
- POST: 提交数据
- PUT: 更新资源
- DELETE: 删除资源
- HEAD: 获取头部信息

**状态码:**

- 1xx: 信息性状态码
- 2xx: 成功状态码(200 OK)
- 3xx: 重定向状态码(301 永久重定向)
- 4xx: 客户端错误(404 Not Found)
- 5xx: 服务器错误(500 Internal Server Error)

## 网络编程

### Socket编程

!!! success "Socket"
    网络通信的端点,应用程序与网络协议栈的接口。

**类型:**

- **流式Socket(SOCK_STREAM)**: TCP
- **数据报Socket(SOCK_DGRAM)**: UDP
- **原始Socket(SOCK_RAW)**: 直接访问IP层

**编程流程:**

```mermaid
graph TB
    A[创建Socket] --> B[绑定地址bind]
    B --> C[监听listen]
    C --> D[接受连接accept]
    D --> E[收发数据recv/send]
    E --> F[关闭连接close]
```

## 网络新技术

### 5G网络

<div style="background-color: #FCE4EC; padding: 15px; margin: 10px 0; border-left: 4px solid #E91E63; border-radius: 5px;">
    <strong>5G网络</strong>
    <p style="margin: 5px 0;">第五代移动通信网络。</p>
</div>

**特点:**

- 高速率: 峰值速率10Gbps
- 低延迟: 端到端延迟1ms
- 大连接: 每平方公里百万级设备
- 高可靠: 99.999%可靠性

**应用场景:**

- 增强移动宽带(eMBB)
- 海量机器类通信(mMTC)
- 超可靠低延迟通信(uRLLC)

### 物联网(IoT)

!!! info "物联网"
    物物相连的互联网,实现万物互联。

**架构:**

- 感知层: 传感器、RFID
- 网络层: 通信网络
- 应用层: 智能应用

**关键技术:**

- 传感器技术
- 嵌入式系统
- 无线通信
- 云计算

### 软件定义网络(SDN)

<div style="background-color: #E0F2F1; padding: 15px; margin: 10px 0; border-left: 4px solid #00BCD4; border-radius: 5px;">
    <strong>软件定义网络</strong>
    <p style="margin: 5px 0;">控制平面与数据平面分离的新型网络架构。</p>
</div>

**特点:**

- 控制集中化
- 网络可编程
- 开放接口
