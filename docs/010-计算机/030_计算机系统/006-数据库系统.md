# 数据库系统

## 概述

数据库系统是具有数据管理功能的计算机系统,是实现有组织地、动态地存储大量关联数据,方便多用户访问的计算机软、硬件资源组成的系统。

!!! note "数据库系统定义"
    数据库系统 = 数据库 + 数据库管理系统(DBMS) + 应用程序 + 数据库管理员(DBA)

## 数据库系统的组成

```mermaid
graph TB
    A[数据库系统] --> B[数据库<br/>Database]
    A --> C[数据库管理系统<br/>DBMS]

    A --> E[数据库管理员<br/>DBA]
    
    C --> C1[数据定义语言DDL]
    C --> C2[数据操纵语言DML]
    C --> C3[数据控制语言DCL]
```

### 1. 数据库

<div style="background-color: #E3F2FD; padding: 15px; margin: 10px 0; border-left: 4px solid #2196F3; border-radius: 5px;">
    <strong>数据库(Database)</strong>
    <p style="margin: 5px 0;">长期存储在计算机内的、有组织的、可共享的数据集合。</p>
</div>

**特点:**

- 持久存储
- 有组织
- 可共享
- 冗余度小
- 独立性高

### 2. 数据库管理系统

<div style="background-color: #E8F5E9; padding: 15px; margin: 10px 0; border-left: 4px solid #4CAF50; border-radius: 5px;">
    <strong>数据库管理系统(DBMS)</strong>
    <p style="margin: 5px 0;">对数据库进行定义、操纵、管理和维护的软件系统。</p>
</div>

**功能:**

- 数据定义
- 数据操纵
- 数据控制
- 数据维护

### 3. 应用程序

<div style="background-color: #FFF3E0; padding: 15px; margin: 10px 0; border-left: 4px solid #FF9800; border-radius: 5px;">
    <strong>应用程序</strong>
    <p style="margin: 5px 0;">利用数据库管理系统访问和操作数据库的程序。</p>
</div>

### 4. 数据库管理员

<div style="background-color: #F3E5F5; padding: 15px; margin: 10px 0; border-left: 4px solid #9C27B0; border-radius: 5px;">
    <strong>数据库管理员(DBA)</strong>
    <p style="margin: 5px 0;">负责数据库系统的设计、维护和管理的人员。</p>
</div>

## 数据模型

!!! tip "数据模型"
    数据模型是数据库系统中用于提供信息表示和操作手段的形式构架。

### 1. 层次模型

<div style="border: 2px solid #4CAF50; padding: 10px; margin: 10px 0; border-radius: 5px;">
    <strong>层次模型</strong>
    <p style="margin: 5px 0;">用树形结构表示实体之间联系的模型。</p>
</div>

**特点:**

- 有且仅有一个根节点
- 除根节点外,其他节点有且仅有一个父节点

### 2. 网状模型

<div style="border: 2px solid #2196F3; padding: 10px; margin: 10px 0; border-radius: 5px;">
    <strong>网状模型</strong>
    <p style="margin: 5px 0;">用图结构表示实体之间联系的模型。</p>
</div>

**特点:**

- 允许一个节点有多个父节点
- 允许多个节点没有父节点

### 3. 关系模型

<div style="border: 2px solid #FF9800; padding: 10px; margin: 10px 0; border-radius: 5px;">
    <strong>关系模型</strong>
    <p style="margin: 5px 0;">用二维表结构表示实体之间联系的模型。</p>
</div>

**特点:**

- 数据结构简单
- 理论基础扎实
- 使用广泛

## 关系数据库

!!! info "关系数据库"
    基于关系模型的数据库。

### 基本概念

#### 1. 关系(表)

<div style="background-color: #FCE4EC; padding: 10px; margin: 10px 0; border-left: 4px solid #E91E63;">
    <strong>关系</strong>
    <p style="margin: 5px 0;">一个二维表,表示实体集。</p>
</div>

#### 2. 元组(行)

<div style="background-color: #E0F2F1; padding: 10px; margin: 10px 0; border-left: 4px solid #00BCD4;">
    <strong>元组</strong>
    <p style="margin: 5px 0;">表中的一行,表示一个实体。</p>
</div>

#### 3. 属性(列)

<div style="background-color: #FFF9C4; padding: 10px; margin: 10px 0; border-left: 4px solid #FFC107;">
    <strong>属性</strong>
    <p style="margin: 5px 0;">表中的一列,表示实体的一个特征。</p>
</div>

### 关系运算

- **选择(Selection)**: 从关系中选取满足条件的元组
- **投影(Projection)**: 从关系中选取若干属性列
- **连接(Join)**: 将两个关系按条件连接

## 数据库事务

!!! info "事务"
    数据库中一组操作的逻辑单元,要么全部成功,要么全部失败。

### ACID特性

<div style="background-color: #E3F2FD; padding: 15px; margin: 10px 0; border-left: 4px solid #2196F3; border-radius: 5px;">
    <strong>ACID特性</strong>
    <p style="margin: 5px 0;">事务的四个基本特性。</p>
</div>

**四个特性:**

- **原子性(Atomicity)**: 事务是不可分割的工作单位
- **一致性(Consistency)**: 事务执行后数据库状态一致
- **隔离性(Isolation)**: 多个事务并发执行互不干扰
- **持久性(Durability)**: 事务一旦提交,永久生效

## 数据库规范化

!!! tip "规范化"
    消除数据冗余,减少更新异常的过程。

### 范式

<div style="background-color: #E8F5E9; padding: 15px; margin: 10px 0; border-left: 4px solid #4CAF50; border-radius: 5px;">
    <strong>范式(Normal Form)</strong>
    <p style="margin: 5px 0;">关系模式满足的不同程度的规范化要求。</p>
</div>

**各级范式:**

- **第一范式(1NF)**: 属性不可再分
- **第二范式(2NF)**: 消除非主属性对码的部分函数依赖
- **第三范式(3NF)**: 消除非主属性对码的传递函数依赖
- **BCNF**: 消除主属性对码的部分和传递函数依赖
- **第四范式(4NF)**: 消除非平凡且非函数依赖的多值依赖

### 规范化过程

```mermaid
graph LR
    A[未规范化] --> B[1NF]
    B --> C[2NF]
    C --> D[3NF]
    D --> E[BCNF]
```

## 数据库索引

!!! success "索引"
    提高数据检索速度的数据结构。

### 索引类型

<div style="background-color: #FFF3E0; padding: 15px; margin: 10px 0; border-left: 4px solid #FF9800; border-radius: 5px;">
    <strong>常见索引类型</strong>
</div>

**类型:**

- **B+树索引**: 最常用的索引结构
    - 支持范围查询
    - 自动排序
    - 适合磁盘存储

- **哈希索引**: 基于哈希表
    - 等值查询快
    - 不支持范围查询
    - 不支持排序

- **全文索引**: 用于文本搜索
    - 支持模糊匹配
    - 支持自然语言搜索

- **空间索引**: 用于地理数据
    - R树索引
    - 支持空间查询

### 索引设计原则

**应该创建索引的情况:**

- 频繁作为WHERE条件的字段
- 频繁用于JOIN的字段
- 频繁用于ORDER BY的字段
- 频繁用于GROUP BY的字段

**不应该创建索引的情况:**

- 频繁更新的字段
- 区分度低的字段(如性别)
- 数据量小的表

## 数据库并发控制

!!! warning "并发控制"
    保证多个事务并发执行时数据库的一致性。

### 封锁协议

<div style="background-color: #F3E5F5; padding: 15px; margin: 10px 0; border-left: 4px solid #9C27B0; border-radius: 5px;">
    <strong>封锁协议</strong>
    <p style="margin: 5px 0;">通过加锁实现并发控制。</p>
</div>

**锁类型:**

- **共享锁(S锁)**: 用于读操作,多个事务可同时加S锁
- **排他锁(X锁)**: 用于写操作,独占锁

**三级封锁协议:**

- **一级封锁协议**: 修改数据前加X锁,防止丢失更新
- **二级封锁协议**: 一级+读取数据前加S锁,防止读脏数据
- **三级封锁协议**: 二级+S锁直到事务结束,防止不可重复读

### MVCC

<div style="background-color: #FCE4EC; padding: 15px; margin: 10px 0; border-left: 4px solid #E91E63; border-radius: 5px;">
    <strong>多版本并发控制(MVCC)</strong>
    <p style="margin: 5px 0;">通过保存数据的多版本来实现并发控制。</p>
</div>

**优点:**

- 读操作不阻塞写操作
- 写操作不阻塞读操作
- 提高并发性能

**实现:**

- 保存数据的多个版本
- 通过版本号或时间戳判断可见性
- 定期清理旧版本

## 数据库恢复

!!! info "数据库恢复"
    在故障发生后将数据库恢复到一致状态。

### 恢复技术

**日志文件:**

- **重做日志(Redo Log)**: 重做已提交事务的修改
- **撤销日志(Undo Log)**: 撤销未提交事务的修改

**检查点(Checkpoint):**

- 定期将内存数据写入磁盘
- 记录检查点位置
- 恢复时从检查点开始

### 恢复策略

```mermaid
graph TB
    A[系统故障] --> B[从检查点开始]
    B --> C[正向扫描日志]
    C --> D[重做已提交事务]
    D --> E[反向扫描日志]
    E --> F[撤销未提交事务]
```

## NoSQL数据库

<div style="background-color: #E0F2F1; padding: 15px; margin: 10px 0; border-left: 4px solid #00BCD4; border-radius: 5px;">
    <strong>NoSQL数据库</strong>
    <p style="margin: 5px 0;">非关系型数据库,适合大数据和Web应用。</p>
</div>

### 类型

**1. 键值存储:**

- Redis、Memcached
- 简单快速
- 适合缓存

**2. 文档存储:**

- MongoDB、CouchDB
- 存储JSON/BSON文档
- 灵活的数据模型

**3. 列族存储:**

- HBase、Cassandra
- 适合海量数据
- 高写入性能

**4. 图数据库:**

- Neo4j、JanusGraph
- 存储图结构数据
- 适合社交网络、推荐系统

### CAP理论

!!! tip "CAP理论"
    分布式系统最多只能同时满足三项中的两项。

**三个特性:**

- **一致性(Consistency)**: 所有节点数据一致
- **可用性(Availability)**: 每个请求都有响应
- **分区容错性(Partition tolerance)**: 网络分区时系统仍能运行

**权衡:**

- CA: 传统关系数据库(单机)
- CP: MongoDB、HBase
- AP: Cassandra、DynamoDB
