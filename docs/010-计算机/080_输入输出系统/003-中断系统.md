# 中断系统

## 概述

!!! note "中断系统"
    中断是CPU暂停当前程序,转去处理紧急事件,处理完后返回继续执行的机制。

## 中断类型

<div style="background-color: #E3F2FD; padding: 15px; margin: 10px 0; border-left: 4px solid #2196F3; border-radius: 5px;">
    <strong>中断分类</strong>
</div>

### 硬中断

!!! tip "硬中断"
    由硬件产生的中断。

- **外部中断**: I/O设备中断
- **内部中断**: 异常、故障

### 软中断

<div style="background-color: #E8F5E9; padding: 15px; margin: 10px 0; border-left: 4px solid #4CAF50; border-radius: 5px;">
    <strong>软中断</strong>
    <p style="margin: 5px 0;">由程序指令产生的中断。</p>
</div>

- **系统调用**: 用户请求系统服务
- **陷阱**: 调试断点

## 中断处理过程

`mermaid
graph TB
    A[中断请求] --> B[中断响应]
    B --> C[保护现场]
    C --> D[中断服务]
    D --> E[恢复现场]
    E --> F[中断返回]

    style A fill:#E3F2FD
    style B fill:#E8F5E9
    style C fill:#FFF3E0
    style D fill:#F3E5F5
    style E fill:#FCE4EC
    style F fill:#E1F5FE
`

### 详细步骤

!!! info "处理步骤"
    1. **中断请求**: 设备发出中断信号
    2. **中断判优**: 选择优先级最高的中断
    3. **中断响应**: CPU响应中断
    4. **保护现场**: 保存PC、PSW等
    5. **中断服务**: 执行中断服务程序
    6. **恢复现场**: 恢复保存的寄存器
    7. **中断返回**: 返回被中断程序

## 中断优先级

<div style="background-color: #FFF3E0; padding: 15px; margin: 10px 0; border-left: 4px solid #FF9800; border-radius: 5px;">
    <strong>优先级原则</strong>
    <ul style="margin: 5px 0;">
        <li>硬件故障 > 程序错误 > 外部中断</li>
        <li>高速设备 > 低速设备</li>
        <li>输入设备 > 输出设备</li>
    </ul>
</div>

## 中断嵌套

!!! warning "中断嵌套"
    高优先级中断可以打断低优先级中断服务程序。
